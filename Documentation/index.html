Installation

We have used Laravel Framework to craft best possible support center :

  • Setting up laravel and other dependencies
    I have written a complete post on how to setup laravel on digitalocean if you are using digitalocean then click here and follow . If you are not using digitalocean then just upload all files from source folder and
  • Permissions
    As stated in laravel installation documentation you need to give write permission to app/storage directory . Other directories/files which require write access are app/views/emails , public/uploads , app/config/mail.php and app/config/packages/bogardo/mailgun/config.php
  • Database
    You can change database settings in app/config/database.php . Change database , username and password in mysql section and save the file . Now run php artisan migrate which will create all tables schema and insert required data . By default admin email is admin@mail.com and password is admin
  • Creating your company
    Xenon support center is crafted to real-world situation . In real-world a company have departments which are managed by department managers/admins , department admins are take care of operators in respective department . So we first have to create a company before we create anything . Login with your default email and password and click on company in sidebar and on next page click on + sign to create a new company . If you are using livechat then company domain should match with the domain which is using livechat , like if you want to put livechat on http://kodeinfo.com then domain should be http://kodeinfo.com
  • Create Department admins and Departments
    Click on departments and click on Create new department admin , fill the fields and click save . Now you have a department admin which is not handling any department . So lets create a new department and assign that department to him . Click on departments and then click on create new department , enter name , select permissions , select department admin and click save . I have listed all permissions in the end of documentation .
  • Setup mail using smtp/mailgun
    You can choose between smtp/mailgun to send mails . Go to settings , if you want to use mailgun check use mailgun and fill all fields then click save . If you want to use smtp then uncheck use mailgun in mailgun settings and save then go to smtp tab and fill all fields and save .
  • Mail Templates using blade/mailchimp
    You can choose between blade/mailchimp templating . Go to settings and then click on mailchimp tab enter mailchimp api key and check "use mailchimp for email templates" and save . If you want to use default templates then just uncheck "use mailchimp" . If you are using mailchimp then go to mailchimp templates in sidebar and click pair emails to template and pair all emails to mailchimp templates . You can use custom variables in template to replace information , at the end of documentation i have shown list of variable available to each email template .

Features

  • Built on Laravel Framework
    Laravel is one of the most popular framework at the moment and is full of features. It is easy to use and it offers a large variety of packages using composer .
  • Live Chat
    It contains livechat which you can integrate on any website . It is fast and powerful , customer can start conversation simply by entering details and click start and operators will be notified . All conversations are saved in closed conversations .
  • Canned Messages
    You can create predefined messages which can be used by operators for quick reply
  • Operators
    You can create operators for company departments . Operators have separate dashboard and can access functionalities which is assigned by department admin .
  • Department Admins and Departments
    You can create department admins who are in-charge of particular departments . You can also create department and assign department admin to it .
  • Mailchimp Templating
    This is a unique feature only visible if use mailchimp is checked in settings . You can create your own templates using mailchimp's powerful editor use variables shown in documentation , then you can pair mailchimp templates to email templates so that next time emails use mailchimp templates instead of default .
  • Permissions ~50 levels
    Now control your department admins/operators using permissions . Complete system is based on permissions like if you assign tickets.create to operator then only he can create tickets . You can control each and everything using permissions from what users edit and what users can see .
  • Tickets
    By default if no operators are available livechat is converted to ticket . Users can login to there accounts and reply to existing tickets or create new tickets , tickets also supports attachments . Tickets panel also shows user details like ip address , country , priority , status etc .
  • Customers
    You can create customers and assign them to companies directly from admin panel .
  • Companies
    You can create companies directly from admin panel .
  • Blocking
    You can block ip using ip blocking . Types of blocking are block from accessing chat , block from creating tickets , block from login , full website blocking .
  • Mailgun
    You can use mailgun to send emails . Just go to settings , check use mailgun and enter details then save .

Email Templates

Below are variables which are available for each template and can be used in mailchimp templates.

  • emails.tickets.ticket_created
    • {{$ticket_id}} will display ticket ID
    • {{$ticket_subject}} will display ticket subject
    • {{$ticket_description}} will display ticket description
    • {{$ticket_status}} will display ticket status in numeric
    • {{$ticket_status_txt}} will display ticket status in text
    • {{$ticket_priority}} will display ticket priority in numeric
    • {{$ticket_priority_txt}} will display ticket priority in text
    • {{$company_name}} will display company name
    • {{$company_description}} will display company description
    • {{$company_domain}} will display company domain
    • {{$company_logo}} will display company logo
    • {{$department_name}} will display department name
    • {{$has_attachment}} will return true if updated message have attachment
    • {{$attachment_path}} will return path of attachment
    • {{$receiver_name}} will display receiver/customer name
    • {{$receiver_email}} will display receiver/customer email
  • emails.tickets.ticket_updated
    • {{$ticket_id}} will display ticket ID
    • {{$ticket_subject}} will display ticket subject
    • {{$ticket_description}} will display ticket description
    • {{$ticket_status}} will display ticket status in numeric
    • {{$ticket_status_txt}} will display ticket status in text
    • {{$ticket_priority}} will display ticket priority in numeric
    • {{$ticket_priority_txt}} will display ticket priority in text
    • {{$company_name}} will display company name
    • {{$company_description}} will display company description
    • {{$company_domain}} will display company domain
    • {{$company_logo}} will display company logo
    • {{$department_name}} will display department name
    • {{$has_attachment}} will return true if updated message have attachment
    • {{$attachment_path}} will return path of attachment
    • {{$updated_message}} will return new updated message for current ticket
    • {{$receiver_name}} will return name of receiver
    • {{$receiver_email}} will return email of receiver
  • emails.users.welcome
    • {{$name}} will display name of new customer
    • {{$email}} will display email of new customer
    • {{$password}} will display password of new customer
    • {{$company_name}} will display company name
    • {{$company_description}} will display company description
    • {{$company_domain}} will display company domain
    • {{$company_logo}} will display company logo
  • emails.users.activate
    • {{$name}} will display name of customer
    • {{$email}} will display email of customer
    • {{$user_id}} will display user id of customer
    • {{$activation_url}} will display url for customer to activate
    • {{$activation_code}} will display activation code for new customer

    • {{$company_description}} will display company description
    • {{$company_domain}} will display company domain
    • {{$company_logo}} will display company logo
  • emails.users.reset_password
    • {{$name}} will display name of customer
    • {{$email}} will display email of customer
    • {{$reset_code}} will display reset code which is used to reset password
    • {{$reset_url}} will display complete url to reset password
  • emails.users.password_changed
    • {{$name}} will display name of customer

Languages

To change language clone app/lang/en and rename folder to new language code , Now you can edit all files inside it . Next is to set this as default language . Go to app/config/app.php and change locale to new language code you just created
